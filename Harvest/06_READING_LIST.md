# 06 — Reading List (the exact OpenFoundry files worth a human read)

> Ordered so a developer builds the wedge's mental model fastest, then sees the gaps. Paths are relative to the OpenFoundry repo root. Each line: **what you'll learn** and the **verdict**. All sizes/line-refs VERIFIED 2026-06-14 (re-`rg` before citing). Total core read ≈ 1 hour.

## Tier 1 — read these (the harvestable shapes, ~30 min)

1. **`services/ontology-service/migrations/20260419100004_initial_ontology.sql`** (71 lines) — **the single most valuable file.** The entire ontology data model in 5 tables: `object_types`, `properties` (note `validation_rules JSONB`), `link_types` (note `cardinality TEXT`), `object_instances` (system-cols + `properties JSONB`), `link_instances` (one unified edge table). *Learn:* the objects-as-JSONB + unified-edge shape we borrow (→ `02`). *Verdict:* **HARVEST** the shape; note the omissions (no version, no tombstone, no value_type table, everything CASCADE).
2. **`services/ontology-service/migrations/20260423113000_action_types.sql`** (17 lines) — the `action_types` table: the `input_schema JSONB` (params) **+** `config JSONB` (operation spec) split, `operation_kind`, `permission_key`, `confirmation_required`. *Learn:* the action definition shape (→ `01`). *Verdict:* **HARVEST** the split; note it's definition-only (no execution/audit/revert tables).
3. **`services/ontology-service/src/handlers/actions.rs`** (1202 lines — skim, don't read every line) — the kinetic core. Read: `HttpInvocationConfig` (L55–62), `ActionPlan` enum (L73–99), `validate_action`/`plan_action` (L959–991, L534–603), `execute_action` (L993–1202). *Learn:* the **validate → preview(dry-run) → execute** contract (HARVEST), and *first-hand* the gaps — no transaction, no version guard, no audit, hard delete, no `CreateObject` variant. *Verdict:* **HARVEST the contract; REJECT the write path.**
4. **`services/ontology-service/src/domain/type_system.rs`** (47 lines) — `VALID_TYPES` (9 property types), `validate_property_value` (base-type match only), `validate_cardinality` (4 values). *Learn:* the concrete enum vocabularies to copy into config; and that `validation_rules` is **never consulted**. *Verdict:* **HARVEST the vocab; note the dead validation.**
5. **`services/audit-service/src/models/audit_event.rs`** (199 lines) — the `AuditEvent` struct (L78–101): `actor/action/resource_type/resource_id/status(success|failure|denied)/severity/classification/metadata/occurred_at` (+ `sequence/previous_hash/entry_hash`). *Learn:* the audit *field shape* to map onto a Supabase audit table (→ `02` §4). *Verdict:* **HARVEST the fields.**
6. **`libs/auth-middleware/src/{claims.rs, row_level_security.rs}`** (~41 + 61 lines) — `Claims { sub, email, roles, permissions, org_id, attributes }` → `RlsContext` via `From<&Claims>`. *Learn:* JWT → actor wiring, the B5 pattern to map onto Supabase Auth (`auth.uid()`/JWT claims). *Verdict:* **HARVEST the pattern**; note `org_filter` builds SQL by string interpolation (don't copy that).
7. **`proto/ontology/ontology.proto`** (3844 bytes — the *only* non-empty ontology proto) — `ObjectType`/`Property`/`LinkType` message field names. *Learn:* vocabulary for the YAML config. *Verdict:* **HARVEST naming only**; note `action.proto`/`object.proto` are **0 bytes** and the codegen never ran.

## Tier 2 — read to understand what's *not* there (the honesty pass, ~15 min)

8. **`libs/query-engine/src/optimizer_rules.rs`** (5 lines) — literally `//! Future: Row-level security filter injection`. *Learn:* the Hasura-style "compile policy into WHERE" is a TODO comment. *Verdict:* **CONFIRMS B6 unbuilt** → use Supabase RLS.
9. **`services/audit-service/src/domain/immutable_log.rs`** (~35 lines) — `chain_hash` is a `format!` string, not a digest. *Learn:* the "tamper-evident hash chain" is fake (→ `05` A5). *Verdict:* **ANTIPATTERN.**
10. **`services/auth-service/src/domain/abac.rs`** + **`handlers/policy_mgmt.rs`** — a real ABAC evaluator that renders a `row_filter`… which `evaluate_policy` then just *returns to the caller* instead of applying. *Learn:* read-side security is advisory only, and built by `{{token}}` substitution. *Verdict:* **ANTIPATTERN** (don't copy the templating).
11. **`services/ontology-service/src/handlers/objects.rs`** (~127 lines) — object CRUD that writes the raw JSONB blob with **zero** validation, and has **no `update_object` route**. *Learn:* "type validation ✅" is untrue at the instance level (→ `05` A3/A10). *Verdict:* **ANTIPATTERN.**
12. **`services/gateway/src/middleware/audit.rs`** — the only wired audit emitter; stamps `actor="system:gateway"`, async/best-effort. *Learn:* why provenance is thin (→ `05` A6). *Verdict:* **ANTIPATTERN.**

## Tier 3 — context only (skim, ~10 min)

13. **`README.md`** (15 KB) + **`ROADMAP.md`** (20 KB) — the *intent*. Read **as marketing**: cross-check every "✅ Done" against code (see `05` A3). `ARCHITECTURE.md`/`CHANGELOG.md`/`CONTRIBUTING.md`/`SECURITY.md` are **0 bytes** — skip.
14. **`.env.example`** (50 lines) — the assumed infra (Postgres + Redis + NATS + S3/MinIO + Meilisearch + Qdrant + JWT). *Learn:* exactly the heavy stack the wedge collapses to one Supabase. *Verdict:* **what NOT to carry.**
15. **`apps/web/`** (SvelteKit 2 / Svelte 5 / Vite 6 / Tailwind; `src/lib/api/client.ts` typed fetch wrapper; ~28 routes, ~14k `.svelte` lines) — a real TS SPA against the Rust `/api/v1`. *Learn:* a typed client + feature-page pattern is feasible for our thin front end (though we'll likely choose React/Next + Supabase client, not Svelte against a separate backend). *Verdict:* **reference only.**
16. **`Cargo.toml` L47** (`license = "Apache-2.0"`) + empty **`LICENSE`** — the license ambiguity (→ `05` A1). *Verdict:* note and move on.

> **Skip entirely** (0-byte scaffolding or platform-scale, see `04`/`05` A2): `domain/{graph,indexer,sync,time_series}.rs`, `domain/search/*`, `handlers/{search,bulk}.rs`, `models/{constraint,interface}.rs`, `libs/audit-trail/*` (empty), all of `services/{ml,ai,geospatial,nexus,marketplace,notebook,streaming,fusion}-service`, `infra/`, `proto/*` except `ontology/ontology.proto`. There is **no `docs/`**, **no worked use-case fixture**, and `libs/testing/src/fixtures.rs` is empty — author Bailey's fixture fresh (see `01`).
