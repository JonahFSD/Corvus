# Domain documentation

This is a single-context repository.

Before exploration, read the root `CONTEXT.md` and any relevant records under `docs/adr/`. Proceed silently if a referenced artifact does not exist.

Use glossary terms in issue titles, specs, tests, review findings, and code. If a needed term is absent, either reconsider the invented language or update the glossary through the domain-modeling workflow.

If proposed work conflicts with an ADR, name the conflict explicitly. Never silently override a recorded decision.
